Cihaz Test Protokolü · QA Sign-off

Görünenin Ötesinde Bir Deniz

Gerçek cihazlarda (Android + iOS tabletler) MVP build'in uçtan uca QA testi için adım-adım protokol. Pilot test öncesi ve her release döngüsü öncesi koşturulur.

Süre~45 dk per cihaz
Test fleet4 kategori önerilir
Sign-offPilot test izni için zorunlu
TekrarHer release öncesi

★ Bu protokol gerçek cihaza ihtiyaç duyar

Unity Editor'da değil, gerçek tablette basılmış AR kartlarla çalıştırılır. Üretici (sen veya QA ekibi) cihaz + kart seti + test çekirdek malzemesi elindeyken uygular.

1Test öncesi hazırlık

1.1 Test fleet

Minimum bir cihaz, ideal olarak 4 farklı kategoride test:

KategoriÖnerilenHedef metrik
Android orta-uçSamsung Galaxy Tab A8, Xiaomi Pad 530 FPS min
Android üst-uçSamsung Galaxy Tab S8, Google Pixel Tablet60 FPS hedef
iOS orta-uçiPad 9. nesil30 FPS min
iOS üst-uçiPad Air (M1+), iPad Pro60 FPS hedef

1.2 Test materyali

  • APK / iOS build cihaza yüklenmiş
  • Tüm 20 basılı kart hazır (production print veya placeholder)
  • Sınıf veya benzeri ortam aydınlatması
  • HDMI / USB-C-to-HDMI kablo (projeksiyon testi için, opsiyonel)
  • Stopwatch (cold start ölçümü)
  • Network monitor (privacy doğrulama, opsiyonel)

1.3 Cihaz state

  • Cihaz tam şarjlı (%90+) — battery drain ölçümü için
  • Önceki uygulamalar kapalı (clean RAM baseline)
  • Wi-Fi açık ama uygulama hiçbir network çağrısı yapmamalı
  • Uygulamanın önceki install state'i temiz

5Privacy stance doğrulama (5 dk)

⊘ Bu adım kritik

Privacy policy'nin doğru olduğunu cihaz seviyesinde teyitle. Tek bir HTTP isteği bile çıkarsa store listing'deki "no data collected" deklarasyonunu ihlal eder.

5.1 Network monitor

  • Bir network monitor app yükle (NetGuard for Android, Charles Proxy)
  • Uygulamayı 5 dk kullan
  • Network log'unda hiçbir HTTP/HTTPS isteği görünmemeli
  • DNS lookup da olmamalı

5.2 Android permission check

  • Settings → Apps → ARFishing → Permissions
  • Sadece Camera permission listede olmalı
  • Storage permission istemiyor olmalı
  • Internet permission yok olmalı — manifest override çalışmış demek

5.3 iOS permission check

  • Settings → ARFishing
  • Sadece Camera access listede
  • Location, Photos, Contacts vb. gözükmemeli

5.4 PlayerPrefs sınırı

  • Editor olmadan uygulamayı kaldırıp tekrar yükle
  • İlk açılışta default state'te olmalı (önceki skor silinmiş)

9Sonraki adım

PASS sonrası

  1. Build artifact'i pilot test cihazlarına yükle
  2. Pilot eğitmenle koordinat (Docs/TeacherGuide.html)
  3. 3 sınıf pilot seansı planla
  4. Pilot feedback toplama protokolü (TeacherGuide §6 + §7)

FAIL sonrası

  1. Bug listesi prioritize et
  2. Severity blocker olan'ları code fix
  3. Re-test full protocol
